<objective>
## Goal
Define the Agent B2 system prompt and build POST `/api/rewrite` — a streaming endpoint that merges the edited design schema (with locked fields respected) and a grammar blueprint into a final image-generation prompt. The prompt text streams to the client via SSE; the completed output is saved to `prompt_outputs` in the `onFinish` callback. The response includes an `X-Output-Id` header so the frontend knows the saved record's ID before the stream ends.

## Purpose
This is the last agent in the pipeline. Agent A extracts structure; Agent B1 learns grammar; Agent B2 synthesises both into one high-quality prompt. Without this route, Phases 5–6 have no prompt to work with. Getting the merge logic right here — especially locked-field handling and the grammar-blueprint-as-style-guide pattern — determines prompt quality for everything downstream.

The streaming approach (vs `generateObject`) is correct here because the output is free text, not structured data, and streaming gives the editor UI progressive feedback for long generation calls (consistent with the streaming decision recorded in STATE.md).

## Output
- `src/lib/schemas/prompt-rewrite.ts` — `B2_REWRITE_SYSTEM_PROMPT` + `buildRewriteInput()` helper
- `src/app/api/rewrite/route.ts` — POST handler: validates → loads schema + blueprint → streams `streamText()` → saves to `prompt_outputs` in `onFinish`
</objective>

## Task 2 — Create `src/lib/schemas/prompt-rewrite.ts`

**File:** `src/lib/schemas/prompt-rewrite.ts`

**Action:** Create the B2 system prompt and the input builder. This file owns:
1. `B2_REWRITE_SYSTEM_PROMPT` — what Agent B2 is and how it must behave
2. `buildRewriteInput()` — serialises schema + blueprint + lockedFields into a structured user message

### B2_REWRITE_SYSTEM_PROMPT
The prompt must instruct the model to:
- Act as a professional prompt engineer, NOT a content describer
- Output ONLY the final image-generation prompt — no preamble, no explanation, no quotation marks
- Follow the grammar blueprint exactly: sequence, compression style, density, characteristic phrases, vocabulary
- For LOCKED sections: reproduce the extracted values verbatim in the prompt; do not reinterpret or summarise
- For UNLOCKED sections: incorporate the values but allow stylistic re-expression consistent with the grammar
- Target length: match the blueprint's `avg_length` character count (±20%)
- Never add platform tags (e.g. `--ar`, `--v`) — those are added by the export panel

Example system prompt:
```
You are Agent B2, a professional image-generation prompt engineer.

Your task: merge a design schema with a grammar blueprint to produce ONE high-quality image-generation prompt.

Rules:
1. Output ONLY the prompt text. No explanation, no preamble, no surrounding quotes.
2. Follow the grammar blueprint's writing style exactly — sequence, compression, vocabulary.
3. Sections marked LOCKED must be reproduced faithfully; do not paraphrase.
4. Sections marked UNLOCKED may be re-expressed in the blueprint's voice.
5. Match the blueprint's average length (±20%).
6. Do not add platform-specific tags (--ar, --v, style tags). The export panel handles those.
```

### buildRewriteInput(schema, blueprint, lockedFields)

Types:
```ts
import type { DesignSchema, GrammarBlueprint } from "@/types/db"
import type { DesignExtraction } from "@/lib/schemas/design-extraction"
import type { GrammarBlueprintExtraction } from "@/lib/schemas/grammar-blueprint"

export function buildRewriteInput(
  schema: DesignSchema,
  blueprint: GrammarBlueprint,
  lockedFields: string[]
): string
```

Implementation: build a structured string with two sections:

**Section 1 — DESIGN SCHEMA:**
Parse each field from the schema JSON columns. For each of the 6 sections (frame, palette, layout, type_scale, text_fields, elements), output:
```
[SECTION NAME] — LOCKED
{JSON.stringify(parsed value, null, 2)}

or:

[SECTION NAME] — unlocked
{JSON.stringify(parsed value, null, 2)}
```

Only include sections where the DB column is non-null.

Helper pattern:
```ts
function schemaSection(
  label: string,
  raw: string | null,
  locked: boolean
): string {
  if (!raw) return ""
  const status = locked ? "LOCKED" : "unlocked"
  return `[${label}] — ${status}\n${raw}\n`
}
```

**Section 2 — GRAMMAR BLUEPRINT:**
```
WRITING GRAMMAR:
Summary: {blueprint.summary from distilled_grammar JSON}
Sequence template: {blueprint.sequence_pattern.template from JSON}
Compression: {blueprint.compression_style}
Density: {blueprint.density} (~{blueprint.avg_length} chars avg)
Characteristic phrases: {characteristic_phrases joined by " | "}
Style vocabulary: {style_vocabulary joined by ", "}
```

Parse `blueprint.distilled_grammar` as `GrammarBlueprintExtraction` to access sub-fields.
Parse `blueprint.sequence_pattern` as `SequencePatternSchema` inferred type.

Use null-safe access throughout — blueprint fields can be null.

## Task 3 — Create `src/app/api/rewrite/route.ts`

**File:** `src/app/api/rewrite/route.ts`

**Action:** Create the streaming POST handler:

```ts
import { randomUUID } from "crypto"
import { z } from "zod"
import { streamText } from "ai"
import { getDb } from "@/lib/db"
import { getTextProvider } from "@/lib/ai"
import { B2_REWRITE_SYSTEM_PROMPT, buildRewriteInput } from "@/lib/schemas/prompt-rewrite"
import { isLocalMode, getTextModel } from "@/lib/env"
import type { DesignSchema, GrammarBlueprint } from "@/types/db"
```

### Request schema
```ts
const RequestSchema = z.object({
  projectId: z.string().min(1),
  blueprintId: z.string().min(1).optional(),
})
```

### Handler steps:

1. **Validate body** — `safeParse`, return 400 on failure (same pattern as distill route)

2. **Verify project exists** — `SELECT id FROM projects WHERE id = ?`, return 404 if missing

3. **Load design schema** — `SELECT * FROM design_schemas WHERE project_id = ? ORDER BY created_at DESC LIMIT 1`
   - Return 400 `{ error: "No design schema found for this project" }` if null

4. **Load grammar blueprint:**
   - If `blueprintId` provided: `SELECT * FROM grammar_blueprints WHERE id = ?` — return 404 if not found
   - If not provided: `SELECT * FROM grammar_blueprints WHERE project_id = ? ORDER BY created_at DESC LIMIT 1` — return 400 `{ error: "No grammar blueprint found" }` if null

5. **Parse locked fields** from `schema.locked_fields` (JSON array string, default `'[]'`)

6. **Pre-generate output ID:**
   ```ts
   const outputId = randomUUID()
   ```

7. **Build rewrite input** via `buildRewriteInput(schema, blueprint, lockedFields)`

8. **Call streamText:**
   ```ts
   const result = streamText({
     model: getTextProvider(),
     system: B2_REWRITE_SYSTEM_PROMPT,
     prompt: rewriteInput,
     onFinish: async ({ text }) => {
       try {
         const db = getDb()
         db.prepare(
           `INSERT INTO prompt_outputs
              (id, project_id, schema_snapshot, blueprint_id, final_prompt, model_used)
            VALUES (?, ?, ?, ?, ?, ?)`
         ).run(
           outputId,
           projectId,
           schema.raw_analysis ? JSON.stringify(schema) : null,
           blueprint.id,
           text,
           getTextModel()
         )
       } catch (err) {
         console.error("[POST /api/rewrite] onFinish DB save failed:", err)
       }
     },
   })
   ```
   Note: `schema_snapshot` stores the full serialised DesignSchema row (JSON) — includes all field columns as-is (they are already TEXT/JSON strings in DB).

9. **Return streaming response with X-Output-Id header:**
   ```ts
   const dataStreamResponse = result.toDataStreamResponse()
   return new Response(dataStreamResponse.body, {
     status: dataStreamResponse.status,
     headers: new Headers({
       ...Object.fromEntries(dataStreamResponse.headers.entries()),
       "X-Output-Id": outputId,
       "Access-Control-Expose-Headers": "X-Output-Id",
     }),
   })
   ```

10. **Catch block** — return `NextResponse.json({ error: "Rewrite failed" }, { status: 500 })` on any non-streaming error. Note: errors that occur after streaming begins will not produce a 500 response — they will cut the stream. Log them with `console.error("[POST /api/rewrite]", error)`.
