<objective>
## Goal
Create the `evaluation_scores` DB table, `EvaluationResultSchema` Zod schema, and `POST /api/evaluate` endpoint so Agent D can score a generated image against its reference across 5 design dimensions and persist the result.

## Purpose
This is the foundation of the refinement loop. Without evaluation data, the user has no structured way to identify where the generated output diverged from the reference. Agent D turns visual comparison into actionable, dimension-specific feedback that the user can edit and feed back into Agent B2.

## Output
- `evaluation_scores` table in SQLite + Supabase schemas
- `EvaluationScore` TypeScript interface in `types/db.ts`
- `src/lib/schemas/evaluation-score.ts` — `DimensionScoreSchema`, `EvaluationResultSchema`, `EVALUATION_PROMPT`
- `src/app/api/evaluate/route.ts` — POST endpoint returning `{ id, scores, critique, iteration }`
</objective>

<task type="auto">
  <name>Task 2: Create EvaluationResultSchema Zod + EVALUATION_PROMPT</name>
  <files>src/lib/schemas/evaluation-score.ts</files>
  <action>
    Create new file `src/lib/schemas/evaluation-score.ts`:

    ```typescript
    import { z } from "zod"

    export const DimensionScoreSchema = z.object({
      score: z.number().int().min(1).max(5)
        .describe("1 = very different, 3 = partial match, 5 = exact match"),
      verdict: z.enum(["match", "partial", "miss"])
        .describe("match = score 4-5, partial = score 2-3, miss = score 1"),
      notes: z.string()
        .describe("Specific observations about this dimension — what matched and what diverged"),
    })

    export const EvaluationResultSchema = z.object({
      composition: DimensionScoreSchema
        .describe("Spatial arrangement, balance, rule of thirds, negative space use"),
      typography: DimensionScoreSchema
        .describe("Typeface classification, weight, case treatment, scale hierarchy"),
      palette: DimensionScoreSchema
        .describe("Color palette character — hues, contrast, saturation, monochromatic vs polychromatic"),
      layout: DimensionScoreSchema
        .describe("Layout structure — single-column, full-bleed, asymmetric split, text overlays"),
      elements: DimensionScoreSchema
        .describe("Visual element types — photography, geometric shapes, textures, rule lines"),
      critique: z.string()
        .describe("One paragraph: overall assessment, most significant divergences, suggested changes"),
    })

    export type EvaluationResult = z.infer<typeof EvaluationResultSchema>
    export type DimensionScore = z.infer<typeof DimensionScoreSchema>

    /** Prompt used by Agent D for structured visual evaluation */
    export const EVALUATION_PROMPT = `You are a senior art director evaluating how closely a generated image matches a reference design.

    You will receive TWO images:
    1. REFERENCE — the original design to match
    2. GENERATED — the AI-generated output to evaluate

    Score the generated image against the reference across 5 dimensions. For each dimension:
    - score: 1 (very different) → 5 (exact match)
    - verdict: "match" (score 4-5), "partial" (score 2-3), "miss" (score 1)
    - notes: specific observations — what is correct, what diverged, how

    DIMENSIONS:
    1. composition — spatial arrangement, subject placement, balance, negative space use, bleed
    2. typography — typeface class (serif/sans subtype), weight, case treatment, scale hierarchy, letter-spacing
    3. palette — color palette character (monochromatic vs polychromatic, hue accuracy, contrast level)
    4. layout — structural layout (single-column, full-bleed, text overlay, asymmetric split)
    5. elements — visual element types present (photography, geometric shapes, textures, rule lines, overlays)

    Be precise. Avoid vague observations like "colors are different" — specify which colors, where, and why it matters.
    critique: one paragraph summarising the most important divergences and what a prompt engineer should change.`
  </action>

<task type="auto">
  <name>Task 3: Create POST /api/evaluate — Agent D endpoint</name>
  <files>src/app/api/evaluate/route.ts</files>
  <action>
    Create `src/app/api/evaluate/route.ts` following the exact pattern of `/api/analyze/route.ts`:

    ```typescript
    import { NextResponse } from "next/server"
    import { z } from "zod"
    import { generateObject } from "ai"
    import { getSupabaseServer } from "@/lib/supabase-server"
    import { getVisionProvider } from "@/lib/ai"
    import { EvaluationResultSchema, EVALUATION_PROMPT } from "@/lib/schemas/evaluation-score"
    import type { EvaluationScore, GeneratedImage, PromptOutput, DesignSchema } from "@/types/db"

    export const maxDuration = 60

    const RequestSchema = z.object({
      prompt_output_id:   z.string().min(1),
      generated_image_id: z.string().min(1),
    })
    ```

    POST handler logic:
    1. Validate body with RequestSchema (400 on fail)
    2. Load `generated_images` row by `generated_image_id` (404 if missing or no url)
    3. Load `prompt_outputs` row by `prompt_output_id` → get `project_id` (404 if missing)
    4. Load latest `design_schemas` for `project_id` ordered by created_at DESC → get `reference_image` (400 if no reference_image)
    5. Extract generated image filename from public URL:
       `const genFilename = new URL(generatedImage.url).pathname.split("/").pop()`
    6. Download both images from Supabase Storage using `.download(filename)`:
       - `referenceBlob` ← `supabase.storage.from("uploads").download(designSchema.reference_image)`
       - `generatedBlob` ← `supabase.storage.from("uploads").download(genFilename)`
       - Convert both to Buffer via `Buffer.from(await blob.arrayBuffer())`
    7. Call `generateObject`:
       ```typescript
       const { object: result } = await generateObject({
         model: getVisionProvider(),
         schema: EvaluationResultSchema,
         messages: [{
           role: "user",
           content: [
             { type: "text", text: "REFERENCE IMAGE (the target design):" },
             { type: "image", image: referenceBuffer },
             { type: "text", text: "GENERATED IMAGE (the AI output to evaluate):" },
             { type: "image", image: generatedBuffer },
             { type: "text", text: EVALUATION_PROMPT },
           ],
         }],
       })
       ```
    8. Count existing evaluations for this prompt_output_id to determine iteration:
       ```typescript
       const { count } = await supabase
         .from("evaluation_scores")
         .select("id", { count: "exact", head: true })
         .eq("prompt_output_id", prompt_output_id)
       const iteration = (count ?? 0) + 1
       ```
    9. Insert row:
       ```typescript
       const { data: scoreRow, error } = await supabase
         .from("evaluation_scores")
         .insert({
           prompt_output_id,
           generated_image_id,
           project_id: promptOutput.project_id,
           reference_image: designSchema.reference_image,
           generated_image_url: generatedImage.url,
           scores: JSON.stringify(result),   // TEXT in SQLite, JSONB in Supabase
           critique: result.critique,
           iteration,
         })
         .select()
         .single()
       if (error) throw error
       ```
    10. Return `NextResponse.json({ id, scores: result, critique: result.critique, iteration })`
    11. Catch block: `console.error("[POST /api/evaluate]", error)` → 500 with `{ error: "Evaluation failed" }`

    Avoid: importing from `src/lib/db.ts` (production uses Supabase, not SQLite)
    Avoid: storing scores as nested JSON-in-JSON — `scores` column receives `JSON.stringify(result)` which is the full EvaluationResult object
  </action>
