<objective>
## Goal
Inject relevant correction lessons into Agent A's analysis prompt via embedding-based
retrieval — so every new project benefits from lessons learned on all prior projects.

## Purpose
Phase 12-01 fills `correction_memories.embedding` after each Agent E correction. Without
retrieval injection, those embeddings are indexed but never queried — the system learns
but doesn't apply what it learned. This plan closes the loop: Agent A now asks "what
have we gotten wrong before on images like this?" before extracting the schema.

## Output
- Supabase: `match_correction_memories` RPC function for cosine similarity retrieval
- `/api/analyze`: 2-pass flow — brief vision description → embed → retrieve top-5 lessons
  → inject as "Prior Extraction Corrections" block → full schema extraction
- Graceful degradation: if no lessons exist (empty DB) or embedding fails, runs as
  standard single-pass analysis with no behavioural change
</objective>

<task type="checkpoint:human-action">
  <what-to-do>
    Create the `match_correction_memories` similarity search function in Supabase.

    Open your Supabase project → SQL Editor → New query → paste and run:

    ```sql
    CREATE OR REPLACE FUNCTION match_correction_memories(
      query_embedding vector(768),
      match_count     int DEFAULT 5
    )
    RETURNS TABLE (
      id         uuid,
      dimension  text,
      lesson     text,
      similarity float
    )
    LANGUAGE sql STABLE
    AS $$
      SELECT
        id,
        dimension,
        lesson,
        1 - (embedding <=> query_embedding) AS similarity
      FROM correction_memories
      WHERE embedding IS NOT NULL
      ORDER BY embedding <=> query_embedding
      LIMIT match_count;
    $$;
    ```

    **Why a function instead of raw ORDER BY:**
    Supabase JS client does not support `ORDER BY embedding <=> $vector` syntax directly
    in `.select()`. An RPC function is the standard Supabase pgvector retrieval pattern.

    **Verify:**
    After running, check: Database → Functions → filter by `match_correction_memories`.
    You should see it listed with return type `record`.

    Alternatively, test it with:
    ```sql
    -- Quick smoke test (all zeros vector — should return rows if correction_memories has data)
    SELECT * FROM match_correction_memories(array_fill(0, ARRAY[768])::vector, 3);
    ```

    ### Insert retrieval block between step 3 and step 4

    Replace the comment `// 4. Run structured extraction via AI SDK` with the following
    block (keep the existing generateObject call intact — only the prompt content changes):

    ```typescript
    // 3b. Retrieve relevant correction lessons via embedding similarity (Phase 12)
    // Brief first-pass: describe typography + palette → embed → cosine query
    // Falls back to single-pass analysis silently if no lessons or embedding fails.
    let priorCorrectionsBlock = ""

    try {
      // Brief description — lightweight vision call, ~5s
      const { text: styleDescription } = await generateText({
        model: getVisionProvider(),
        messages: [
          {
            role: "user",
            content: [
              { type: "image", image: imageBuffer },
              {
                type: "text",
                text: "In 1-2 sentences, describe the dominant typographic style " +
                      "(typeface classification, stroke contrast, weight) and color " +
                      "palette character of this image. Be specific about visual signals.",
              },
            ],
          },
        ],
      })

      const queryVector = await computeEmbedding(styleDescription)

      if (queryVector) {
        const { data: lessons } = await supabase.rpc("match_correction_memories", {
          query_embedding: `[${queryVector.join(",")}]`,
          match_count:     5,
        })

        if (lessons && (lessons as { dimension: string; lesson: string }[]).length > 0) {
          const lessonLines = (lessons as { dimension: string; lesson: string }[])
            .map((l) => `• [${l.dimension}] ${l.lesson}`)
            .join("\n")

          priorCorrectionsBlock =
            `PRIOR EXTRACTION CORRECTIONS — from similar past images:\n` +
            `Apply these lessons when you detect the same visual patterns:\n\n` +
            lessonLines +
            `\n\n---\n\n`
        }
      }
    } catch (retrievalErr) {
      // Non-fatal — log and continue with standard single-pass analysis
      console.error("[analyze] lesson retrieval failed (non-fatal):", retrievalErr)
    }

    // 4. Run structured extraction via AI SDK
    ```

    ```typescript
    content: [
      { type: "image", image: imageBuffer },
    ```
