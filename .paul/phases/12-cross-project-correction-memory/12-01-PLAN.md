<objective>
## Goal
Compute `text-embedding-004` embeddings for each Agent E lesson immediately after
`correction_memories` rows are inserted, and persist them to the `embedding vector(768)`
column. Create a HNSW vector index in Supabase to enable cosine similarity retrieval.

## Purpose
Phase 11 leaves `correction_memories.embedding` as NULL after every refine cycle.
Without embeddings, Phase 12-02 cannot do cosine similarity retrieval — lessons are
dark data. This plan lights them up: every future Agent E correction immediately
becomes searchable by semantic similarity, enabling cross-project learning in Plan 12-02.

## Output
- `src/lib/embeddings.ts` — `computeEmbedding(text): Promise<number[] | null>` helper
- `src/app/api/refine/route.ts` — correction_memories insert gains `.select()` + parallel
  embedding compute + UPDATE; timing stays well within maxDuration = 60
- Supabase: HNSW index on `correction_memories(embedding vector_cosine_ops)`
</objective>

    ### 1a. Create `src/lib/embeddings.ts`

    New file — export a single async function:

    ```typescript
    import { env } from "@/lib/env"

    /**
     * Compute a text-embedding-004 vector (768 dims) for the given text.
     * Returns null on any error — callers treat embedding as non-fatal.
     * Uses the same GEMINI_API_KEY as Gemini image generation.
     */
    export async function computeEmbedding(text: string): Promise<number[] | null> {
      try {
        const res = await fetch(
          `https://generativelanguage.googleapis.com/v1beta/models/text-embedding-004:embedContent?key=${env.GEMINI_API_KEY}`,
          {
            method: "POST",
            headers: { "Content-Type": "application/json" },
            body: JSON.stringify({
              model:   "models/text-embedding-004",
              content: { parts: [{ text }] },
            }),
          }
        )
        if (!res.ok) {
          console.error("[embeddings] API error", res.status)
          return null
        }
        const data = (await res.json()) as { embedding?: { values?: number[] } }
        return data.embedding?.values ?? null
      } catch (err) {
        console.error("[embeddings] computeEmbedding failed:", err)
        return null
      }
    }
    ```

    AVOID:
    - Do not add a new env var — GEMINI_API_KEY is already present in env.ts
    - Do not import from @ai-sdk/* — text-embedding-004 is called via direct REST
      (same pattern as gemini image generation in /api/generate/route.ts)

    ### 1b. Update `src/app/api/refine/route.ts` — section 6e only

    Add import at top of file:
    ```typescript
    import { computeEmbedding } from "@/lib/embeddings"
    ```

    Replace the existing section 6e (correction_memories insert) with:

    ```typescript
    // 6e. Write correction_memories rows + compute embeddings (Phase 12)
    if (correction.lessons.length > 0) {
      const memoryRows = correction.lessons.map((lesson) => ({
        project_id,
        evaluation_score_id,
        dimension:     lesson.dimension,
        lesson:        lesson.lesson,
        bad_value:     lesson.bad_value,
        correct_value: lesson.correct_value,
        // embedding computed below after insert
      }))

      const { data: insertedMemories, error: memoryError } = await supabase
        .from("correction_memories")
        .insert(memoryRows)
        .select("id, lesson")

      if (memoryError) {
        console.error("[refine] correction_memories insert failed:", memoryError)
      } else if (insertedMemories) {
        // Compute embeddings in parallel — non-fatal, awaited before return
        await Promise.allSettled(
          insertedMemories.map(async (row) => {
            const vector = await computeEmbedding(row.lesson as string)
            if (!vector) return
            const { error: embErr } = await supabase
              .from("correction_memories")
              .update({ embedding: `[${vector.join(",")}]` })
              .eq("id", row.id as string)
            if (embErr) {
              console.error("[refine] embedding update failed for", row.id, embErr)
            }
          })
        )
      }
    }
    ```

    KEY DETAILS:
    - `.select("id, lesson")` — only fetch columns needed; avoids pulling vector back
    - `embedding: \`[${vector.join(",")}]\`` — pgvector expects a string in `[f1,f2,...]`
      format when sent via PostgREST (not a raw JS array)
    - `Promise.allSettled` — awaited so Vercel doesn't cut the function before UPDATEs land;
      `.allSettled` (not `.all`) means one embedding failure doesn't cancel the others
    - Timing: 5 lessons × ~500ms parallel = ~500ms extra. Total route stays ~35s

    AVOID:
    - Do not use `Promise.all` — a single embedding failure would reject the whole batch
    - Do not `void` the Promise — the Vercel function must stay open until UPDATEs complete
    - Do not change any other section of the refine route (steps 1–5, 7–11 unchanged)
    - Do not change the `memoryRows` shape — only add `.select("id, lesson")` and the post-insert embedding block

    Run the HNSW index migration in the Supabase SQL editor.

    Open your Supabase project → SQL Editor → New query → paste and run:

    ```sql
    CREATE INDEX IF NOT EXISTS correction_memories_embedding_idx
    ON correction_memories USING hnsw (embedding vector_cosine_ops)
    WITH (m = 16, ef_construction = 64);
    ```

    **Why HNSW over IVFFlat:**
    IVFFlat requires a minimum number of rows for training (`lists` parameter).
    HNSW builds incrementally — works on an empty table and performs better at
    small-to-medium dataset sizes. m=16 / ef_construction=64 are standard defaults.

    **Verify:**
