# Plan 09-01 — Extraction Upgrade

## Goal

Enrich `DesignExtractionSchema` and `DESIGN_ANALYSIS_PROMPT` so Agent A produces semantically precise output: per-role typographic fingerprints, bleed/crop awareness, and palette character vocabulary. Eliminates the dominant quality gap (vague typeface descriptions, missed bleed, misleading palette mood).

## Scope

All changes are in `src/lib/schemas/design-extraction.ts` (schema + prompt) with downstream UI fixes in `analyze-tab.tsx` and `prompt-tab.tsx` to match the new `type_scale` shape.

**No Supabase migration needed** — `type_scale`, `frame`, `palette` are JSONB columns; new shape accepted automatically.

## Acceptance Criteria

- [ ] AC-1: `TypeFingerprintSchema` defined with all 19 fields from spec (core, inferred, spacing, case/align, decorative, numbers, variable, layout-interactions)
- [ ] AC-2: `type_scale` restructured to `z.array(TypeScaleEntrySchema).nullable()` — each entry has `role: string` + `fingerprint: TypeFingerprintSchema`
- [ ] AC-3: `FrameSchema` extended with `crop_treatment` (enum), `composition_notes` (string), `bleed` (boolean)
- [ ] AC-4: `PaletteSchema` extended with `character` (string, free-text palette personality)
- [ ] AC-5: `DESIGN_ANALYSIS_PROMPT` rewritten — instructs classification, stroke contrast, editorial style, bleed/crop detection, palette character
- [ ] AC-6: `analyze-tab.tsx` type_scale section updated — iterates `type_scale` array, shows role label + fingerprint fields
- [ ] AC-7: `prompt-tab.tsx` type_scale section updated — read-only display of per-role fingerprints (lock/unlock at role level)
- [ ] AC-8: `pnpm typecheck` exits 0
- [ ] AC-9: Manual test — run 2–3 reference images through `/api/analyze` locally; schema output shows `fingerprint.classification`, `fingerprint.strokeContrast`, `frame.bleed`, `palette.character` populated

### Task 1 — Add `TypeFingerprintSchema` to design-extraction.ts

**File:** `src/lib/schemas/design-extraction.ts`

Add before `TypeScaleSchema`:

```typescript
export const TypeFingerprintSchema = z.object({
  // Core (Figma panel — visually detectable)
  fontFamily:     z.string().nullable().describe("Identified typeface name if recognisable, else null"),
  fontStyle:      z.string().describe("Weight/style descriptor: 'Thin'|'Light'|'Regular'|'Medium'|'SemiBold'|'Bold'|'ExtraBold'|'Black'"),
  fontWeight:     z.number().int().min(100).max(900).describe("Numeric weight 100–900"),
  fontSize:       z.enum(["display", "large", "medium", "small", "caption"]).describe("Relative size role in composition"),

  // Inferred (Agent A adds — not a Figma field, derived from visual analysis)
  classification: z.enum([
    "high-contrast-serif", "low-contrast-serif", "slab-serif",
    "geometric-sans", "humanist-sans", "grotesque-sans",
    "monospace", "display", "script", "decorative",
  ]).describe("Typeface classification"),
  strokeContrast: z.enum(["none", "low", "medium", "high", "extreme"]).describe("Thin-to-thick stroke ratio"),
  editorialStyle: z.string().describe("Free-text editorial character, e.g. 'Didot-class fashion', 'Swiss International', 'warm humanist'"),

  // Spacing
  letterSpacing:  z.enum(["very-tight", "tight", "normal", "wide", "very-wide"]),
  lineHeight:     z.enum(["compressed", "tight", "normal", "loose", "open"]),

  // Case / Alignment
  case:           z.enum(["none", "uppercase", "lowercase", "title", "small-caps"]),
  alignment:      z.enum(["left", "center", "right", "justified"]),

  // Decorative
  decoration:     z.array(z.string()).describe("e.g. ['underline'], ['strikethrough'], or []"),

  // Numbers (null when no numerals visible)
  numberStyle:    z.enum(["lining", "old-style"]).nullable(),
  numberPosition: z.enum(["normal", "superscript", "subscript"]).nullable(),

  // Variable font axes (null when not a variable font or not detectable)
  variable: z.object({
    weight: z.number().nullable(),
    slant:  z.number().nullable(),
  }).nullable(),

  // Layout interactions
  hangingPunctuation: z.boolean().describe("Punctuation hangs outside the text block"),
  paragraphIndent:    z.boolean().describe("First line of paragraphs is indented"),
  listStyle:          z.enum(["none", "unordered", "ordered"]),
})

export type TypeFingerprint = z.infer<typeof TypeFingerprintSchema>
```

### Task 5 — Rewrite `DESIGN_ANALYSIS_PROMPT`

**File:** `src/lib/schemas/design-extraction.ts`

Replace the existing prompt with a richer version that explicitly instructs:
- Typeface classification vocabulary (not just "serif/sans-serif")
- Stroke contrast assessment
- Editorial style labelling
- Per-role fingerprint population
- Bleed/crop detection
- Palette character (not just mood)

```typescript
export const DESIGN_ANALYSIS_PROMPT = `Analyze the visual design of this image. Extract its structural and aesthetic properties with the precision of a senior art director. Focus on composition, color relationships, typographic treatment, and visual element arrangement. Do NOT describe subject matter or narrative — describe the design language.

TYPOGRAPHY — for each distinct text role (headline, subheading, body, caption, label, eyebrow, etc.):
- Classify the typeface precisely: not just "serif" but "high-contrast-serif" (Didot/Bodoni class), "low-contrast-serif" (Garamond class), "slab-serif", "geometric-sans" (Futura/Avenir), "humanist-sans" (Gill Sans/Frutiger), "grotesque-sans" (Helvetica/Akzidenz), "monospace", "display", "script", or "decorative"
- Assess stroke contrast: none (monolinear), low, medium, high, or extreme (hairline-to-bold jump)
- Describe editorial style in free text: e.g. "Didot-class high fashion", "Swiss International", "warm humanist editorial", "brutalist grotesque"
- Infer numeric weight (100–900) and style descriptor
- Note letter-spacing and line-height in relative terms
- Detect case treatment, text alignment, and any decorative elements
- Note hanging punctuation, paragraph indentation, and list style if visible
- Set numberStyle/numberPosition if numerals are present; null otherwise
- Set variable axes if it appears to be a variable font (unusual weight interpolation); null otherwise

FRAME:
- Detect bleed: true if any element (text, image, colour) extends to or bleeds off a frame edge
- Describe crop treatment: none / tight-crop / bleed-crop / letterbox / full-frame
- Note composition observations beyond the structure field (edge tension, text overflow, layering)

PALETTE:
- Sample actual pixel hex values for primary, secondary, accent, background, text_color
- Describe palette character in design terms — not "vibrant" but "high-contrast monochromatic", "warm duotone", "editorial black-and-white", "pastel editorial", "bold primary triad"
- Set mood and saturation as before

ELEMENTS: list design components (photography, shapes, textures, overlays, rules, geometric elements) — not scene objects.

Be precise and specific. Vague descriptions like "some colors", "bold font", or "various elements" are not acceptable.`
```
