  <files>
    src/app/projects/[id]/analyze-tab.tsx
  </files>
  <action>
    Create a `"use client"` component with this state machine:
    - `idle` — shows upload widget (drop zone + file input)
    - `uploading` — shows spinner ("Uploading image…")
    - `ready` — shows image preview + "Analyze Design" button
    - `analyzing` — shows spinner ("Analyzing design…") + disabled button
    - `done` — shows SchemaViewer with extracted schema + "Upload new image" reset button
    - `error` — shows error message + "Try again" button (resets to idle)

    **Props:** `{ projectId: string }`

    **State shape:**
    ```typescript
    type Phase = "idle" | "uploading" | "ready" | "analyzing" | "done" | "error"
    // Use useState with an object or separate states — either is fine
    // Track: phase, filename, previewUrl, schema (DesignExtraction), errorMsg
    ```

    **Upload widget (idle state):**
    - A `<div>` acting as drop zone: handle `onDragOver` (preventDefault), `onDrop` (prevent default, take first file)
    - Hidden `<input type="file" accept="image/*">` triggered by clicking the drop zone
    - Visual: dashed border, centered text "Drop an image here or click to browse"
    - On file selected: generate preview with `URL.createObjectURL(file)`, call upload()

    **Upload function:**
    ```typescript
    async function upload(file: File) {
      setPhase("uploading")
      const fd = new FormData()
      fd.append("file", file)
      const res = await fetch("/api/upload", { method: "POST", body: fd })
      if (!res.ok) { setPhase("error"); setErrorMsg("Upload failed"); return }
      const data = await res.json() as { filename: string }
      setFilename(data.filename)
      setPhase("ready")
    }
    ```

    **Analyze function:**
    ```typescript
    async function analyze() {
      setPhase("analyzing")
      const res = await fetch("/api/analyze", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ projectId, filename }),
      })
      if (!res.ok) { setPhase("error"); setErrorMsg("Analysis failed"); return }
      const data = await res.json() as { id: string; schema: DesignSchema }
      // Parse JSON TEXT fields from DB row into objects for display
      setSchema(data.schema)
      setPhase("done")
    }
    ```

    **Important — DB returns JSON as TEXT strings:** The `DesignSchema` from the API has fields like `frame`, `palette`, etc. as JSON TEXT strings (per the DB design decision). Parse them before display:
    ```typescript
    const frame = JSON.parse(data.schema.frame ?? "null") as DesignExtraction["frame"] | null
    const palette = JSON.parse(data.schema.palette ?? "null") as DesignExtraction["palette"] | null
    // etc. for all fields
    ```
    Store parsed fields separately, or build a helper `parsedSchema` object from the response.

    **SchemaViewer (inline in this file or as a separate const):**
    Render each section as a Card with a header and field rows. Sections:

    1. **Style Summary** — full-width text block at the top (most prominent)
    2. **Frame** — 4 fields in a 2-col grid: aspect_ratio, orientation, composition, negative_space
    3. **Palette** — 5 color rows (label + swatch + hex) + mood + saturation. Color swatch: `<span style={{ backgroundColor: hex }} className="inline-block h-4 w-6 rounded border" />`
    4. **Layout** — structure, focal_point, depth, visual_hierarchy (as a bulleted list)
    5. **Typography** — if `type_scale` is null, show "No typography detected". Otherwise: primary_typeface, weight_range, scale, letter_spacing
    6. **Text Fields** — if empty, show "No text elements detected". Otherwise: list each item (role, style, position) as a compact row
    7. **Elements** — list each VisualElement (type, description, prominence badge, position)

    Use existing shadcn components: `Card`, `CardHeader`, `CardContent`, `Badge`, `Button`.
    Use Tailwind for layout (grid, gap, text sizing).

    **Avoid:**
    - External drag-drop libraries — native HTML events only
    - Storing the File object in state beyond the upload call
    - Calling `URL.revokeObjectURL` synchronously (let browser GC handle it for now)
    - Any server-side imports — this is `"use client"` only
