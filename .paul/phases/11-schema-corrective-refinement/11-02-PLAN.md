<objective>
## Goal
Update the Refine flow in output-tab.tsx to match the new /api/refine response shape
({ prompt_output_id, iteration } — no url), calling /api/generate as a second step,
with distinct loading labels for each step.

## Purpose
Plan 11-01 removed the internal /api/generate call from the refine route to stay within
the 60s Vercel timeout. The UI must chain the two calls explicitly. Without this fix,
clicking "Refine" silently fails because data.url is undefined.

Also fixes a latent bug: after a refine cycle, activeOutput.id was not updated to the
new prompt_output_id, causing the next Evaluate to look up the wrong generated image.

## Output
- output-tab.tsx: refine() is 2-step, EvalPhase gains a visible sub-label,
  activeOutput.id updates to new prompt_output_id after refine completes
</objective>

    2. Rewrite the `refine()` function body:

       ```typescript
       async function refine() {
         if (!evalData?.id) return
         setEvalPhase("refining")
         setRefineStep("correcting")
         setEvalError("")
         try {
           // Step 1: Schema correction + prompt rebuild
           const refineRes = await fetch("/api/refine", {
             method: "POST",
             headers: { "Content-Type": "application/json" },
             body: JSON.stringify({
               project_id:          projectId,
               evaluation_score_id: evalData.id,
             }),
           })
           if (!refineRes.ok) {
             const ct = refineRes.headers.get("content-type") ?? ""
             const err = ct.includes("application/json")
               ? (await refineRes.json()) as { error?: string }
               : { error: `Server error ${refineRes.status}` }
             throw new Error(err.error ?? "Refinement failed")
           }
           const refineData = (await refineRes.json()) as {
             prompt_output_id: string
             iteration:        number
           }

           // Step 2: Image generation from corrected prompt
           setRefineStep("generating")
           const genRes = await fetch("/api/generate", {
             method: "POST",
             headers: { "Content-Type": "application/json" },
             body: JSON.stringify({ outputId: refineData.prompt_output_id }),
           })
           if (!genRes.ok) {
             const ct = genRes.headers.get("content-type") ?? ""
             const err = ct.includes("application/json")
               ? (await genRes.json()) as { error?: string }
               : { error: `Server error ${genRes.status}` }
             throw new Error(err.error ?? "Image generation failed")
           }
           const genData = (await genRes.json()) as { id: string; url: string; provider: string }

           // Update active output: new id (for next Evaluate), new image_url, new provider
           setOutputs(prev =>
             prev.map((o, i) =>
               i === activeIndex
                 ? { ...o, id: refineData.prompt_output_id, image_url: genData.url, image_provider: genData.provider }
                 : o
             )
           )
           setIteration(refineData.iteration)
           setRefineStep(null)
           setEvalPhase("idle")
           setEvalData(null)
           setCritiqueText("")
         } catch (e) {
           setRefineStep(null)
           setEvalError(e instanceof Error ? e.message : "Refinement failed")
           setEvalPhase("error")
         }
       }
       ```
