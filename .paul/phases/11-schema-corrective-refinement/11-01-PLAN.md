# Plan 11-01: Agent E + Schema-Corrective Refine Route

**Phase:** 11 — Schema-Corrective Refinement Loop
**Plan:** 11-01 of 2
**Status:** 🔲 Pending approval
**Created:** 2026-04-02

---

## Goal

Replace the prompt-patching refine route with a critique-driven schema correction loop.
When evaluation identifies failing dimensions (score ≤ 2), Agent E re-examines the reference
image, outputs corrected schema fields for those dimensions only, and distills transferable
lessons into `correction_memories` — the feed table for Phase 12's embedding retrieval.
The refine route then rebuilds the full prompt from the corrected schema.

## Boundaries

**In scope:**
- `src/lib/schemas/schema-correction.ts` — NEW: SchemaCorrectionSchema + SCHEMA_CORRECTION_PROMPT
- `src/app/api/refine/route.ts` — full rewrite: Agent E + schema fork + B2 (no internal generate)
- `src/lib/schema.postgres.sql` — add `correction_memories` table + `CREATE EXTENSION vector`
- `src/types/db.ts` — add `CorrectionMemory` interface

**Out of scope:**
- UI changes (Plan 11-02)
- Embedding computation (Phase 12)
- Agent A injection of retrieved memories (Phase 12)
- Any changes to evaluate/route.ts, analyze/route.ts, or output-tab.tsx

## Phase 11→12 Data Contract

The `correction_memories` table is the Phase 12 feed table. Phase 11 populates:
- `lesson` — the text unit Phase 12 will embed via `text-embedding-004` (768 dims)
- `bad_value` / `correct_value` — diagnostic context
- `embedding` — created as `vector(768) NULL`; Phase 12 fills it and adds the IVFFlat index

The `SchemaCorrectionSchema.lessons` field is the forward-compatibility hook.
Each `lesson` string must be written by Agent E in a form that embeds well:
lead with the visual signal, state the correct classification, generalise to similar images.

```typescript
import { z } from "zod"
import {
  TypeScaleEntrySchema,
  FrameSchema,
  PaletteSchema,
  LayoutSchema,
} from "@/lib/schemas/design-extraction"

export const EVAL_DIMENSIONS = [
  "composition", "typography", "palette", "layout", "elements",
] as const
export type EvalDimension = typeof EVAL_DIMENSIONS[number]

export const LessonSchema = z.object({
  dimension: z.enum(EVAL_DIMENSIONS),
  lesson: z
    .string()
    .describe(
      "Transferable observation for future projects. Lead with the visual signal. " +
      "Example: 'When letterforms show hairline strokes at junction points against " +
      "thick vertical stems, classify as Didone (high-contrast-serif), not script.'"
    ),
  bad_value:     z.record(z.unknown()),
  correct_value: z.record(z.unknown()),
})

export const SchemaCorrectionSchema = z.object({
  corrected_dimensions: z.array(z.enum(EVAL_DIMENSIONS)).describe(
    "Exactly the dimensions you are correcting — subset of the failing dimensions provided"
  ),
  lessons: z.array(LessonSchema).describe(
    "One lesson per corrected field — stored for cross-project retrieval in Phase 12"
  ),
  // Only populate the fields that map to failing dimensions:
  //   typography  → type_scale
  //   palette     → palette
  //   composition → frame
  //   layout      → layout
  //   elements    → (no dedicated field; handled via critique in fast path)
  type_scale: z.array(TypeScaleEntrySchema).nullable().optional(),
  palette:    PaletteSchema.partial().optional(),
  layout:     LayoutSchema.partial().optional(),
  frame:      FrameSchema.partial().optional(),
})

export type SchemaCorrection = z.infer<typeof SchemaCorrectionSchema>
export type Lesson           = z.infer<typeof LessonSchema>

export const SCHEMA_CORRECTION_PROMPT = `You are Agent E, a design schema correction specialist.

A design extraction agent previously analysed the REFERENCE IMAGE below and produced a structured schema.
A visual evaluation agent then compared the schema-driven output against the reference and identified
FAILING DIMENSIONS — fields where the extraction was inaccurate.

YOUR TASKS:
1. Re-examine the REFERENCE IMAGE carefully — focus on the failing dimensions listed.
2. Output corrected schema values for those dimensions only.
3. For each corrected field, write one LESSON: a transferable observation that prevents the
   same mistake on similar images in future projects.

LESSON FORMAT (follow this precisely):
- Lead with the VISUAL SIGNAL: what is visible in the image that indicates the correct value
- Explain why the current extraction is wrong
- State the correct classification using the exact vocabulary from the schema
- Write it so it applies to SIMILAR images — not just this one

Good lesson example:
"Letterforms in DTMS exhibit extreme thick-thin stroke contrast with hairline serifs at junction
points — the defining signature of Didone typefaces (Bodoni, Didot). The current classification
'script' is wrong because script faces have continuous curved strokes from a single drawing tool.
Correct classification: high-contrast-serif. Key visual check: hairline horizontal serifs on
vertical stems with no bracketing."

OUTPUT ONLY the failing dimensions listed. Do not correct dimensions that are not failing.`
```

### Task 3 — `src/app/api/refine/route.ts` (full rewrite)

**New flow:**

```
1. Validate { project_id, evaluation_score_id, blueprint_id? }
2. Load evaluation_score → parse scores (JSONB → EvaluationResult)
3. Identify failingDimensions: DIMENSIONS.filter(d => scores[d].score <= 2)
4. Load latest design_schema for project
5. Load reference image public URL via getPublicUrl()
6. --- AGENT E BRANCH (failing dimensions exist) ---
   a. Build currentValues context for failing dimension fields only
   b. generateObject(getVisionProvider(), SchemaCorrectionSchema, [
        text: SCHEMA_CORRECTION_PROMPT,
        text: "FAILING DIMENSIONS:\n" + failingDimensions.map(...).join('\n'),
        text: "CURRENT SCHEMA VALUES:\n" + JSON.stringify(currentValues),
        text: "USER CRITIQUE:\n" + evaluationScore.critique,
        image: new URL(refImageUrl),
        text: "Output corrected schema values and lessons for the failing dimensions above."
      ])
   c. Fork schema: INSERT new design_schemas row with corrections merged in
   d. Insert correction_memories rows (one per lesson, embedding = NULL)
   e. Use forked schema for B2
7. --- FAST PATH (all scores >= 3) ---
   Use existing schema unchanged
8. Load grammar blueprint
9. buildRewriteInput(schema, blueprint, lockedFields)
   + append REFINEMENT FEEDBACK section (critique) — keep in both paths
10. generateText (Agent B2) → save prompt_outputs row
11. Return { prompt_output_id, iteration }
    NOTE: no /api/generate call — Plan 11-02 moves generation to the UI
```

**Helper functions** (define above POST handler, not exported):

```typescript
// Parse JSONB field safely (Supabase returns JSONB as object, TEXT as string)
function parseField(raw: unknown): unknown {
  if (raw === null || raw === undefined) return null
  if (typeof raw === "string") {
    try { return JSON.parse(raw) } catch { return null }
  }
  return raw  // already parsed (JSONB from Supabase)
}

// Build context object with only the schema fields that map to failing dimensions
function buildFailingContext(
  schema: DesignSchema,
  failing: readonly EvalDimension[]
): Record<string, unknown> {
  const ctx: Record<string, unknown> = {}
  if (failing.includes("typography"))                              ctx.type_scale = parseField(schema.type_scale)
  if (failing.includes("palette"))                                 ctx.palette    = parseField(schema.palette)
  if (failing.includes("composition") || failing.includes("layout")) {
    ctx.frame  = parseField(schema.frame)
    ctx.layout = parseField(schema.layout)
  }
  if (failing.includes("elements")) {
    ctx.text_fields = parseField(schema.text_fields)
    ctx.elements    = parseField(schema.elements)
  }
  return ctx
}

// Merge Agent E corrections into schema field values (returns Supabase INSERT payload)
function mergeCorrectedFields(
  base: DesignSchema,
  correction: SchemaCorrection
): Record<string, unknown> {
  const merged: Record<string, unknown> = {
    project_id:      base.project_id,
    frame:           base.frame,
    palette:         base.palette,
    layout:          base.layout,
    text_fields:     base.text_fields,
    type_scale:      base.type_scale,
    elements:        base.elements,
    style_checksum:  base.style_checksum,
    locked_fields:   base.locked_fields,
    raw_analysis:    base.raw_analysis,
    reference_image: base.reference_image,
  }
  if (correction.type_scale !== undefined) {
    merged.type_scale = JSON.stringify(correction.type_scale)
  }
  if (correction.palette !== undefined) {
    const cur = (parseField(base.palette) ?? {}) as Record<string, unknown>
    merged.palette = JSON.stringify({ ...cur, ...correction.palette })
  }
  if (correction.layout !== undefined) {
    const cur = (parseField(base.layout) ?? {}) as Record<string, unknown>
    merged.layout = JSON.stringify({ ...cur, ...correction.layout })
  }
  if (correction.frame !== undefined) {
    const cur = (parseField(base.frame) ?? {}) as Record<string, unknown>
    merged.frame = JSON.stringify({ ...cur, ...correction.frame })
  }
  return merged
}
```

## Risk Notes

- **EvaluationScore.scores type**: `db.ts` types it as `string | null` but Supabase returns JSONB columns as parsed objects. Cast via `as unknown as EvaluationResult`. Do not JSON.parse — it's already parsed.
- **Agent E timeout**: Single reference image URL + smaller SchemaCorrectionSchema output. Expected ~15-20s. Combined with B2 (~12s) = ~30-35s total — within 60s limit.
- **FrameSchema.partial()**: All fields required in FrameSchema. `.partial()` makes them optional for the correction payload — merge back into the full frame object in `mergeCorrectedFields`.
